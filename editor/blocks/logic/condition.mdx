## Operadores

<ResponseField name="Igual a">
  Corresponderá se o valor fornecido for estritamente igual ao valor.
</ResponseField>

<ResponseField name="Diferente">
  Corresponderá se o valor fornecido for diferente do valor.
</ResponseField>

<ResponseField name="Contém">
  Corresponderá se o valor fornecido contiver o valor. Se uma lista for fornecida, ela corresponderá se a lista tiver pelo menos um elemento em comum com o valor.
</ResponseField>

<ResponseField name="Não contém">
  O mesmo que `Contém`, mas corresponderá ao inverso.
</ResponseField>

<ResponseField name="Maior que">
  Corresponderá se o valor fornecido for maior ou igual ao valor.
</ResponseField>

<ResponseField name="Menor que">
  Corresponderá se o valor fornecido for menor ou igual ao valor.
</ResponseField>

<ResponseField name="Está definido">
  Corresponderá se o valor fornecido não for nulo nem indefinido e não for uma string vazia.
</ResponseField>

<ResponseField name="Está vazio">
  Corresponderá se o valor fornecido for nulo, indefinido ou uma string vazia.
</ResponseField>

<ResponseField name="Começa com">
  Corresponderá se o valor fornecido começar com o valor.
</ResponseField>

<ResponseField name="Termina com">
  Corresponderá se o valor fornecido terminar com o valor.
</ResponseField>

<ResponseField name="Corresponde ao regex">
O valor deve começar e terminar com `/` e conter um padrão regex válido.

Exemplo:

- `/^hello$/` corresponderá se a string for estritamente igual a "hello".
- `/hello/` corresponderá se a string contiver "hello". Como "hello world".
- `/hello/i` corresponderá se a string contiver "hello" sem diferenciação entre maiúsculas e minúsculas. Como "Hello world".
- `/[0-9]+/` corresponderá se a string contiver um ou mais dígitos. Como "123".

</ResponseField>

<ResponseField name="Não corresponde ao regex">
  O mesmo que `Corresponde ao regex`, mas corresponderá se o valor fornecido não corresponder ao padrão regex.
</ResponseField>
